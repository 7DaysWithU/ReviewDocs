# 海明码

## 🧠 一、海明码的核心思想（先理解，再记忆）

### ✅ 目标

> 在传输信息时，**检测并纠正单个比特错误**。

比如你要发 `1010`，希望即使某一位翻转成 `1011`，接收方也能发现并自动纠正。

### ✅ 核心原理

- **用校验位覆盖不同位置的信息位**
- 每个校验位负责一组特定的位置
- 通过校验结果的组合，**唯一确定出错位置**

👉 这就像在超市里，每个商品贴了多个标签，标签上的数字表示它属于哪些区域。如果某个标签错了，就能根据错误模式反推出是哪个商品出问题。

---

## 🔍 二、一步步拆解你的图中步骤

我们以你给的例子为准：信息位 `1010`，要构造海明码。

---

### ✅ 步骤1：确定总长度 —— $2^k \geq n + k + 1$

- $n = 4$（信息位数）
- 要找最小的 $k$，使得 $2^k \geq 4 + k + 1$
- 尝试：
  - $k=3$: $2^3 = 8 \geq 4+3+1 = 8$ → ✅ 成立
- 所以总共 $7$ 位：$4$ 个信息位 + $3$ 个校验位

📌 记住这个公式：**校验位数 $k$ 是满足 $2^k \geq n+k+1$ 的最小整数**

---

### ✅ 步骤2：安排位置 —— 校验位放 $2^i$ 位

| 位号 | 7 | 6 | 5 | 4 | 3 | 2 | 1 |
|------|---|---|---|---|---|---|---|
|     | H₇ | H₆ | H₅ | H₄ | H₃ | H₂ | H₁ |

→ 校验位放在 $2^0=1$, $2^1=2$, $2^2=4$ → 即 **H₁, H₂, H₄**

其余是信息位：H₃, H₅, H₆, H₇

📌 **为什么是 $2^i$ 位？**

因为这些位置的二进制表示只包含一个1（如 1=001, 2=010, 4=100），方便后续用**二进制编码**定位错误。

---

### ✅ 步骤3：校验位计算 —— 哪些位被哪个校验位“覆盖”

> **关键：每个校验位 Pᵢ 负责所有二进制位号中第 i 位为1的位**

例如：

| 位号 | 二进制 | 是否被 P₁(第1位) 覆盖？ | 是否被 P₂(第2位) 覆盖？ | 是否被 P₄(第3位) 覆盖？ |
|------|--------|-------------------------|-------------------------|-------------------------|
| 1    | 001    | ✅                      | ❌                      | ❌                      |
| 2    | 010    | ❌                      | ✅                      | ❌                      |
| 3    | 011    | ✅                      | ✅                      | ❌                      |
| 4    | 100    | ❌                      | ❌                      | ✅                      |
| 5    | 101    | ✅                      | ❌                      | ✅                      |
| 6    | 110    | ❌                      | ✅                      | ✅                      |
| 7    | 111    | ✅                      | ✅                      | ✅                      |

所以：

- **P₁** 覆盖：位号为奇数的位（1,3,5,7）→ H₁, H₃, H₅, H₇
- **P₂** 覆盖：位号中第2位为1的（2,3,6,7）→ H₂, H₃, H₆, H₇
- **P₄** 覆盖：位号中第3位为1的（4,5,6,7）→ H₄, H₅, H₆, H₇

📌 **记住口诀**：
> 校验位 $\text{P}_i$ 覆盖所有二进制位号中第 $i$ 位为 $1$ 的位。
>
> 假如 $3$ 号位出错，对应的二进制为 $011\text{B}$，要求 $\text{P}_1=1,\text{P}_2=1,\text{P}_3=0$，则 $\text{P}_3\text{P}_2\text{P}_1=011\text{B}=3$

---

### ✅ 步骤4：计算校验位值（偶校验）

- 用**异或（⊕）** 实现偶校验：让被覆盖的位中1的个数为偶数

例如：

- $P_1 = H_1 \oplus H_3 \oplus H_5 \oplus H_7$ → 但 $H_1 = P_1$，所以：
  $$
  P_1 = D_1 \oplus D_2 \oplus D_4 \quad (\text{因为 } H_3=D_1, H_5=D_2, H_7=D_4)
  $$

同理：

- $P_2 = D_1 \oplus D_3 \oplus D_4$
- $P_4 = D_2 \oplus D_3 \oplus D_4$

📌 **关键点**：校验位是信息位的异或和，目的是让每组“覆盖集合”中1的个数为偶数。

---

### ✅ 步骤5：纠错 —— 校验方程与错误定位

接收后，重新计算校验位，得到**校验综合** S₁, S₂, S₃：

- $S_1 = P_1 \oplus D_1 \oplus D_2 \oplus D_4$
- $S_2 = P_2 \oplus D_1 \oplus D_3 \oplus D_4$
- $S_3 = P_4 \oplus D_2 \oplus D_3 \oplus D_4$

如果无错 → $S_1=S_2=S_3=0$

如果有错 → $S_1S_2S_3$ 组成一个**二进制数**，对应出错位号！

例如：

- 若 $S_1=0, S_2=1, S_3=0$ → 二进制 `010` → 第 **2** 位出错

📌 **这就是“邪门”的核心机制**：  
> 错误位置的二进制表示，就是校验综合的结果！

---

## 🧩 三、如何记忆？—— 变“邪门”为“巧妙”

### ✅ 记忆口诀（建议背下来）

> **“校验位在 2 的幂次，覆盖按位对齐，异或求偶，综合定位。”**

分解一下：

| 口诀 | 含义 |
|------|------|
| **校验位在 2 的幂次** | 放在 1,2,4,... 位置 |
| **覆盖按位对齐** | 第 i 个校验位覆盖二进制位号中第 i 位为1的位 |
| **异或求偶** | 每组异或，使1的个数为偶数 |
| **综合定位** | 接收后算出 S₁S₂S₃，其二进制值就是出错位号 |

---

### ✅ 图形记忆法（Venn 图）

看你的图中那个三个圆相交的图：

- 每个圆代表一个校验位（P₁,P₂,P₄）
- 每个信息位在多个圆中（表示被多个校验位覆盖）
- 出错时，只有受影响的校验位报错 → 组合起来就是错误位置

👉 这个图很直观！记住它，你就懂了“为什么能定位”。

---

### ✅ 举个简单例子加深印象

信息位：`1010`

- 信息位：D₁=1, D₂=0, D₃=1, D₄=0
- 校验位：
  - $P_1 = D_1 \oplus D_2 \oplus D_4 = 1\oplus0\oplus0 = 1$
  - $P_2 = D_1 \oplus D_3 \oplus D_4 = 1\oplus1\oplus0 = 0$
  - $P_4 = D_2 \oplus D_3 \oplus D_4 = 0\oplus1\oplus0 = 1$

→ 海明码：`P₁ P₂ D₁ P₄ D₂ D₃ D₄` = `1 0 1 1 0 1 0`

即：**1011010**

## 🎯 总结：海明码不邪门，只是设计精妙

| 看似“邪门”的地方 | 实际含义 |
|------------------|----------|
| 校验位在 $2^i$ 位 | 方便用二进制编号定位 |
| 分组方式奇怪 | 每个校验位覆盖特定二进制位 |
| 异或运算 | 实现偶校验，保证奇偶性 |
| 校验综合等于出错位号 | 因为错误影响的校验位正好对应该位的二进制表示 |

⚠️**根本原理：**

**因为我们要加 $k$ 位校验位，而二进制 $k$ 位最多表示 $2^k-1$ 种情况，所以为了覆盖，就需要 $2^k-1\geq n+k$。因为这 $k$ 位校验位能覆盖所有位，所以采用一种与状态压缩根出同源的方法——二进制压缩，正好这 $k$ 个校验位每位代表一个二进制位的 $1$，如`100`，`010`，`001`。再接着采用分组的思想，每个校验位负责自己这一位是 $1$ 的数据位，这样就构成一个组，在这个组里异或完成偶校验，校验结果就是校验位的值。验证也很简单，如果偶校验出现 $1$ 显然就是出错了。最妙的是根据校验位纠错！！！因为是用的二进制压缩，只要出现 $1$ 就表示这一个组出错，$P_i$ 代表的这一个二进制需要亮起，$P_1$ 一直到 $P_k$ 从低位到高位拼起来这就正好是一个 $k$ 位的二进制数，该二进制数正好表明数据中第几位出错(对照[步骤 $3$ 的表格](./ep2_hamming_code.md#✅-步骤3校验位计算--哪些位被哪个校验位覆盖))**

✅**最终建议：这样记忆**

1. **先理解核心思想**：用校验位覆盖信息位，通过异或实现偶校验，错误位置由校验综合决定。
2. **记住两个公式**：
   - $2^k-1 \geq n + k$
   - 校验位 Pi 覆盖位号中第 i 位为1的位
3. **背口诀**：“校验在幂，覆盖对齐，异或求偶，综合定位”
4. **画 Venn 图**，帮助理解覆盖关系
5. **多练几个例子**，直到能快速写出海明码
