    > 题目要求"任何情况都连通"，即无论怎么选边去连，图都是连通的。这时候不能用 $n-1$ 的生成树连法，虽然这是最少的边，但同样 $n-1$ 条边，只要不按照生成树的连法去连，图仍然不连通(如 $4$ 个顶点，结果 $3$ 条边连了个三角形，一个顶点还是孤立的)。这时候应在非连通图最大边数的基础上加一，这样无论怎么连都是连通图，即边数为 $C^2_{n-1}+1$
    >
    > ```Text
    > 1——2
    > | /
    > 3  4
    > ```
    >
    > 如图 $4$ 个顶点 $3$ 条边，不按生成树连法连，导致图不连通

    >     for (int i = 0; i < n; i++)
    >         printf("%d ", arr[i]);
    > }
    > ```
    >
    > 使用`int arr[]`看起来能像普通变量一样使用，不增加记忆负担，但本质上这是个数组指针，只不过没有显式声明出来——使用时也无需显式操作指针地址。**仅有的代价就是应该同时传入数组长度`n`的参数**

    > 当每个结点的工作量相等时，可以认为时间复杂度 $=$ 递归树的结点数 $\times$ 单次函数除递归外的时间复杂度

    ```C
    typedef struct MGraph{

        char Vertices[MAXV];
        int edge[MAXV][MAXV];
    } MGraph;

    bool func(MGraph G){
        bool visit[G.numVertices];
        int indegree[G.numVertices];

        // 统计入度
        for (int i=0;i<G.numVertices;i++){
            visit[i]=false;
            indegree[i]=0;
            for (int j=0;j<G.numVertices;j++)
                indegree[i]+=G.edge[j][i];
        }

        // 逐步找到 k 个入度为 0 的点
        for (int k=0;k<G.numVertices;k++){
            int count=0;
            int in0=-1;

            // 统计入度为 0 的点的个数
            for (int i=0;i<G.numVertices;i++){
                if (!visit[i] && indegree[i]==0){
                  count++;
                  in0=i;
                }
            }

            // 有环或由多个入度为 0 的点
            if(count!=1)
                return false;
            visit[in0]=true;

            // 消除入度为 0 的点对应的出边
            for (int i=0;i<G.numVertices;i++){
                if(G.edge[in0][i]!=0)
                  indegree[i]--;
            }
        }

        return true;
    }
    ```

  * ❌27(确定进程的页框数时需要考虑的因素)
  
    > ***最小物理块数的确定***：
    >
    > 一个显而易见的事实是，随着为每个进程所分配的物理块数的减少，进程在执行过程中的缺页率将会上升，从而会降低进程的执行速度。为使进程能有效地工作，应为它分配一定数目的物理块，但这并不是最小物理块数的概念。
    >
    > **最小物理块数是指，能保证进程正常运行所需的最小物理块数，当系统为进程分配的物理块数少于此值时，进程将无法运行。** 进程应获得的最小物理块数与计算机的硬件结构有关，取决于指令的格式、功能和寻址方式。对于某些简单的机器，若是单地址指令，且采用直接寻址方式，则需的最小物理块数为2。其中，一块是用于存放指令的页面，另一块则是用于存放数据的页面。如果该机器允许间接寻址，则至少要求有3个物理块。对于某些功能较强的机器，其指令长度可能是两个或多于两个字节，因而其指令本身有可能跨两个页面，且源地址和目标地址所涉及的区域，也都可能跨两个页面。正如前面所介绍的在缺页中断机构中要发生6次中断的情况一样，对于这种机器，至少要为每个进程分配6个物理块，以装入6个页面。

    > * $\text{Cache}$ 索引即地址中表示 $\text{Cache}$ 组号的部分
    > * 往年考察 $\text{Cache}$ 缺失率时，数组元素都是从主存块的起始处开始存放的，**但本题数组的起始地址是在主存块的中间，所以需要额外计算数组存放在几个主存块中**
    > * 设 $\text{Cahce}$ 命中时间为 $k_1$ 个时钟周期，缺失损失为 $k_2$ 个时钟周期，$\text{Cache}$ 缺失率为 $p$，则数组元素的平均访问时间为
    >
    >   $$
    >   \text{平均访问时间}=\underbrace{(1-p)k_1}_{\textcolor{Cyan}{命中}}+\underbrace{p(k_1+k_2)}_{\textcolor{Cyan}{未命中}}=k_1+pk_2
    >   $$
    >
    >   计算数组元素的平均访问时间不用再乘元素个数，因为最终也会在分母上求平均除掉，直接用 $\text{Cache}$ 缺失率 $p$ 计算即可。**特别注意，缺失情况不仅产生缺失损失，也存在命中时间，因为处理完缺失后还是得读取**
