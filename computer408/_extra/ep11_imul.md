# 手算乘法

## 1. 无符号数乘法

无符号数乘法的手算方式与十进制竖式乘法完全类似，只是基数为2。

**计算步骤：**

1. 将乘数和被乘数以二进制形式写出
2. 从乘数的最低位开始，逐位与被乘数相乘
3. 每位相乘的结果左移相应的位数（第i位左移i位）
4. 将所有部分积相加得到最终结果

**示例：** 13 × 11 = 143

```Text
     1101  (13)
   × 1011  (11)
   --------
     1101  (第0位：1101×1)
    11010  (第1位：1101×1，左移1位)
   000000  (第2位：1101×0，左移2位)
  1101000  (第3位：1101×1，左移3位)
  --------
 10001111  (143)
```

## 2. 带符号数乘法（补码表示）

带符号数乘法主要有两种方法：

* 方法一：符号绝对值法（推荐手算使用）

  **计算步骤：**
  
  1. **分离符号位**：记录结果的符号（异或运算）
  2. **取绝对值**：将两个操作数都转换为正数
  3. **无符号乘法**：对绝对值进行无符号乘法
  4. **恢复符号**：根据符号位确定最终结果的符号

* 方法二：补码直接乘法（Booth算法）

  这种方法直接在补码形式下运算，但手算较复杂。

## 3. 实例分析：B052H × 0008H

让我们用**符号绝对值法**来计算：

### 步骤1：分析操作数

* B052H = 1011 0000 0101 0010₂ (补码)
* 0008H = 0000 0000 0000 1000₂ (补码)

### 步骤2：确定符号位

* B052H：符号位 = 1（负数）
* 0008H：符号位 = 0（正数）
* 结果符号 = 1 ⊕ 0 = 1（负数）

### 步骤3：取绝对值

* B052H的绝对值：对补码再求补

  ```Text
  1011 0000 0101 0010          //原补码
  0100 1111 1010 1101          //按位取反 
  0100 1111 1010 1110 = 4FAEH  //加1
  ```

* 0008H的绝对值：0008H

### 步骤4：无符号乘法（4FAEH × 0008H）

用竖式计算：

```Text
       0100 1111 1010 1110  (4FAEH = 20398)
     × 0000 0000 0000 1000  (8)
     --------------------------
       0000 0000 0000 0000  (第0-2位为0)
   010 0111 1101 0111 0     (第3位，左移3位)
    --------------------------
   010 0111 1101 0111 0000  (结果左移3位)
= 0010 0111 1101 0111 0000
= 27D70H = 163184
```

验证：20398 × 8 = 163184 ✓

### 步骤5：恢复符号

由于结果符号为负，需要对绝对值乘积求补：

```Text
0010 0111 1101 0111 0000 = 27D70H //绝对值结果
1101 1000 0010 1001 0000 = D8290H //求结果负数的补码
```

**最终结果：D8290H，截断低 $n$ 位为 8290H，发生溢出**

## 4. 普适性说明

这种方法**完全具有普适性**，原因在于：

1. **适用于任意乘数**：不依赖于乘数是否为2的幂次
2. **明确的算法步骤**：符号处理 → 绝对值乘法 → 符号恢复
3. **可手工执行**：只需要基本的二进制运算能力
4. **与硬件实现原理一致**：许多处理器实际采用类似的思路

## 5. 关键要点总结

| 乘法类型 | 核心方法 | 适用场景 |
|---------|----------|----------|
| 无符号乘法 | 直接二进制竖式 | 所有无符号整数 |
| 带符号乘法 | 符号绝对值法 | 手算、理解原理 |
| 带符号乘法 | Booth算法 | 硬件实现、高效计算 |

**考试建议：** 遇到带符号数乘法时，优先使用**符号绝对值法**，因为它：

* 思路清晰，易于理解和验证
* 不容易出错
* 适用于所有情况，不依赖特殊乘数

这种方法确保了你能够处理任何乘法问题，无论乘数是否是2的幂次。
